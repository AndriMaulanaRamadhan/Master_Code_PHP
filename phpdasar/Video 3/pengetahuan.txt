Client side vs server side scripting
Client side : ketika membuat website menggunakan html, css dan javascript itu disebut client side scripting karena semua pemrosesannya terjadi disisi client atau dikomputer diri sendiri masing-masing
Server side : saat website kalian digunakan bahasa PHP di dalamnya maka yang terjadi adalah pada saat membuka websitenya lewat browser maka browser tersebut mengirimkan sebuah request ke web server, maka sekarang prosesnya tidak hanya disisi client tetapi juga disisi server

macam-macam web server :
Apache
IIS
nginx
tomcat
lighttpd

bahasa yang prosesnya disisi server :
PHP
ASP
Ruby
Phyton

cara kerja server side :
jika tidak ada sama sekali bahasa PHP yang akan terjadi adalah web browser akan meminta request ke server dan di server akan membaca apakah dihalaman tersebut ada bahasa php atau tidak, kalau tidak maka web server akan merespon kembali permintaannya
ketika dihalaman tersebut ada sintaks php, maka yang akan terjadi adalah browser request ke server lalu server mengecek apakah ada bahasa php atau tidak, kalau ada maka php akan mengeksekusi script tadi, dan akan dicetak sebagai html lalu dikembalikan lagi sebagai respon ke clientnya lagi, dan itulah bahasa php tidak bisa dicontek karena semua pemrosesannya hanya terjadi di server dan hanya akan ditampilkan html
script tadi bisa terhubung ke database sehingga websitenya menjadi dinamis
